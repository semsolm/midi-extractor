## 프론트엔드 폴더 상세 설명
___

```public/```
* ```index.html```: 모든 React 컴포넌트가 렌더링될 뼈대 HTML 파일입니다.

* ```favicon.ico```: 브라우저 탭에 표시될 아이콘입니다.

```src/``` (Source) 프론트엔드 개발의 핵심이 되는 소스 코드 폴더입니다.

```assets/```: 정적인 자원들을 관리합니다.

* ```css/```: 전역적으로 사용될 CSS 파일(예: global.css, reset.css)을 보관합니다.

* ```images/```: 로고, 배경 이미지 등 프로젝트에서 사용할 이미지 파일들을 보관합니다.

```components/```: 재사용이 가능한 작은 UI 단위(컴포넌트)들을 관리합니다.

* ```common/```: Button.js, Spinner.js, Modal.js 처럼 여러 페이지에서 공통적으로 쓰이는 범용 컴포넌트들을 만듭니다.

* ```layout/```: Header.js, Footer.js, Sidebar.js 와 같이 페이지의 전체적인 레이아웃을 구성하는 컴포넌트들을 배치합니다.

```hooks/```: 반복되는 로직을 재사용하기 위한 커스텀 훅(Custom Hooks)을 관리합니다. 예를 들어, API 요청의 로딩, 데이터, 에러 상태를 관리하는 useApi.js 같은 훅을 만들 수 있습니다.

```pages/```: 실제 사용자가 보게 될 페이지 단위의 컴포넌트들을 관리합니다.

```MainPage.js```: 파일 업로드 UI가 있는 메인 페이지입니다.

```ResultPage.js```: 분석이 완료된 후 악보와 다운로드 버튼을 보여주는 결과 페이지입니다.

```services/```: 백엔드 API와의 통신을 책임지는 로직을 모아둡니다.

```api.js```: 백엔드 서버로 파일을 업로드하고, 결과를 조회하는 등의 모든 네트워크 요청 함수들을 이곳에 정의합니다. 이렇게 하면 나중에 API 주소가 바뀌더라도 이 파일만 수정하면 됩니다.

```App.js```: 애플리케이션의 최상위 컴포넌트로, 페이지 라우팅(경로 설정)을 관리합니다.

```index.js```: React 애플리케이션을 시작하는 진입점 파일입니다.
